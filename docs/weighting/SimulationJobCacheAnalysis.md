# Simulation Job Cache Analysis — TRROUTING_MISSING_DATA

## Problem

When running a network design simulation job (evolutionary algorithm), trRouting reports
`TRROUTING_MISSING_DATA` for every trip. The trRouting log (port 14000) shows:

```
missing nodes cache files!
missing dataSources cache files!
missing agencies cache files!
missing lines cache files!
missing paths cache files!
TransitData loading had error, object will not be valid
```

The same problem occurs on the main trRouting (port 4000).

## Root Cause

**The main cache directory has never been populated with `.capnpbin` files (or they were deleted).**

`prepareCacheDirectory()` copies from the main cache to the job cache.
`fileManager.copyFileAbsolute()` returns `false` silently when the source file does not exist.
Therefore, the job cache ends up missing agencies, nodes, paths, dataSources, and the collection-level lines.capnpbin.

The main cache is populated by:
1. **`STARTUP_RECREATE_CACHE=true`** on server start → calls `recreateCache()` in `dbToCache.ts`
2. **"Recreate cache" button** in the UI
3. **TransitObjectsDataHandler** when saving transit objects individually

Currently `STARTUP_RECREATE_CACHE=false`, and it appears no cache recreation has been triggered
since the project was set up on this branch.

## Changes between `evolutionaryTransitNetworkDesign` and `implementNodeWeighting`

### Files changed that affect the simulation flow

| File | Change | Impact on cache? |
|------|--------|-----------------|
| `TransitNetworkDesignJobWrapper.ts` | Added `applyNodeWeightsIfPresent()` to `loadServerData()`, added `onProgress` param | **No** — only adds weight data to node objects in memory |
| `EvolutionaryTransitNetworkDesignJob.ts` | Added `await` to `this.prepareCacheDirectory()` | **No** — the method is synchronous, `await` on void is a no-op |
| `TransitNetworkJobController.ts` | Refactored job creation: `{ description, ...parameters }` destructuring, added save/replay/lineWeights features | **Possible** — see below |
| `OdTripSimulation.ts` | Removed `driving` from `routingModes`, comments cleanup | **No** — routing modes don't affect cache |
| `services.socketRoutes.ts` | Added standalone node weighting, pause/resume, line weights | **No** — new handlers, don't change job creation flow |

### Key difference in job creation

**Old branch** (`createAndEnqueueEvolutionaryTransitNetworkDesignJob`):
```typescript
data: { parameters: jobParameters }
// jobParameters = full TransitNetworkJobConfigurationType
```

**New branch** (`createEvolutionaryTransitNetworkDesignJob`):
```typescript
const { description, ...parameters } = jobParameters;
data: { parameters, description }
// parameters = TransitNetworkJobConfigurationType minus description
```

This splits `description` out of `parameters`. If any downstream code reads
`parameters.description` it would now be `undefined`. But this should NOT affect
cache generation.

---

## Conclusion

The code changes between the two branches **do not directly break cache generation**.
The root cause is that the main project cache was never populated (or was cleared),
and `prepareCacheDirectory()` silently fails when copying missing source files.

**This is a pre-existing fragile design**: the simulation depends on the main cache
being populated by an external action (startup flag, UI button, or object save),
but there is no guard or fallback.

---

## Suggested Investigation / Fix

### Quick fix (manual)
Set `STARTUP_RECREATE_CACHE=true` in `.env` and restart the server.
Or use the "Recreate cache" button in the UI (if available).

### Robust fix (code)
In `_loadAndPrepareData()` or `prepareCacheDirectory()`, check if the main cache
has the required files. If any are missing, call `recreateCache()` from `dbToCache.ts`
before copying:

```typescript
import { recreateCache } from '../../../capnpCache/dbToCache';

prepareCacheDirectory = async () => {
    const mainCacheDirectory = `${fileManager.directoryManager.cacheDirectory}/${config.projectShortname}`;
    const requiredFiles = ['agencies.capnpbin', 'nodes.capnpbin', 'paths.capnpbin',
                           'dataSources.capnpbin', 'lines.capnpbin', 'services.capnpbin'];
    const cacheIsMissing = requiredFiles.some(
        f => !fileManager.fileExistsAbsolute(`${mainCacheDirectory}/${f}`)
    );
    if (cacheIsMissing) {
        console.warn('Main cache is incomplete, regenerating from database...');
        await recreateCache({ refreshTransferrableNodes: false, saveLines: true });
    }
    // ... existing copy logic ...
};
```

This would make the simulation self-healing and remove the dependency on external
cache population.
